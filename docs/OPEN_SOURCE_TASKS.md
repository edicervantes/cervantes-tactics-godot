# Open Source Tasks

These tasks are useful next steps for collaborators and for keeping the project visibly active.

## Good First Issues

- Add two or three in-game screenshots to the README.
- Improve README run instructions for Windows users.
- Add a small troubleshooting section for Godot import issues.
- Review UI text for clarity and consistency.
- Add comments to the smoke-test scripts explaining what each test validates.

## Gameplay Tasks

- Extract more tactical rule logic from `scripts/battle.gd` into focused systems.
- Improve pathfinding edge cases around blockers and props.
- Add clearer enemy intent previews.
- Improve feedback for traps, defense, healing, and limit attacks.
- Add playtest notes for Chapter I difficulty.

## Maintenance Tasks

- Add a GitHub Actions workflow for Godot import validation.
- Add a smoke-test runner script for local contributors.
- Document which assets are original, third-party, or private-only.
- Keep `CONTENT_LICENSE.md` and `ASSET_LICENSES.md` synchronized as assets are added.

## Public Project Signals

- Tag stable public milestones.
- Keep issues small and beginner-friendly.
- Mark known bugs clearly.
- Use roadmap issues to show active maintenance.
