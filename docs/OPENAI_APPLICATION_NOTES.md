# OpenAI Codex for Open Source Application Notes

This document summarizes the public repository for a potential Codex for Open Source application.

## Repository

https://github.com/edicervantes/cervantes-tactics-godot

## Project Summary

Cervantes Tactics is a Godot 4 tactical RPG prototype with a playable Chapter I vertical slice. The project combines a hybrid 3D diorama battlefield, turn-based tactical rules, data-driven units, save slots, smoke tests, and open-source production documentation.

## Maintainer Role

Primary maintainer: Erick Cervantes.

The maintainer owns the project direction, story canon, public repository, licensing policy, and ongoing roadmap.

## Why This Repository Qualifies

- Public Godot 4 tactical RPG prototype with a playable vertical slice.
- Clear open-source scope with code under MIT.
- Explicit licensing split for story/canon, original art/audio, and third-party assets.
- Data-driven gameplay architecture using Godot resources.
- Smoke-test scripts for menu, battle UI, visual capture, transitions, and save state.
- Public roadmap, contribution guide, issue templates, and release notes.

## How Codex Would Help

Codex would be useful for:

- refactoring tactical rules out of `scripts/battle.gd`;
- improving smoke-test coverage;
- triaging issues and turning playtest notes into focused tasks;
- documenting Godot architecture for new contributors;
- preparing releases and maintenance updates;
- reviewing pull requests for regressions in battle flow, UI, and data resources.

## Suggested Form Text

Cervantes Tactics es un prototipo publico en Godot 4 que muestra como crear un RPG tactico mantenible con agentes: vertical slice jugable, arquitectura data-driven, smoke tests, roadmap, issues y licencias claras. Puede servir como referencia abierta para desarrolladores indie que quieren usar Codex en flujos reales de juego: refactor, QA, documentacion y mantenimiento continuo.

## Notes On AI-Assisted Development

Avoid describing the repository as "almost 100% vibe coded" in the application. A stronger framing is that it is human-directed and AI-assisted: the maintainer owns the creative direction, code review, licensing, publishing, and roadmap, while Codex is used as a development partner for implementation, refactoring, QA, documentation, and maintenance.
